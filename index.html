Overview

Below is our current standard version 0.0.9 of the draft, based of json-schema.org draft 0.4. The project attempts to tackel many problems at once.

  • This should be an easy project, make a point here
  • If I had another point I would make it here

Why JSON?

We feel the JSON is mature enough for doing schematics, which YAML doesn't throughly support yet(though you can easily convert your resume between yaml and json anyway).

Versioning

We will generally do 2 year cycles of revisions to the specification. Though upon release the lifecycle will be short for 3 months whilst last minute revisions are made to the standard

Open Source

Everything on the jsonresume.org domain name is opened source under a MIT license. This is meant to be a community effort to better the HR ecosystem.

Getting Started

We have built a CLI(command line interface) which currently is supported on OS/Linux/Windows via NPM. To start creating your own resume, install the resume-cli via NPM.

sudo npm install -g resume-cli

Excellent! Now initialize your first resume.js

resume init

Exporting

The command line tool, uses an ecosytem of modules we have open sourced to convert your resume.

resume export resume.pdf

resume export resume.html

resume export resume.txt

Publishing

We have also built in hosted support into the command line app. Much like NPM's registry, you can register an account and publish to our servers. We also allow you to password protect and/or publish your resume anonymously. To get started simply type;

resume publish
